# finance-engine-v2

Self-improving crypto strategy lab. Strategies are **data** (typed JSON expression graphs), not code —
generated by genetic programming + Claude Opus ideation, compiled to vectorized TypeScript, and forced
through a statistical gauntlet with absolute floors before they may touch even paper money.

**Read `docs/SPEC.md` first** — it explains why the predecessor failed and what every gate is for.

## Stack

| Piece | Where |
|---|---|
| Dashboard | Next.js on Vercel |
| System of record | Convex (own deployment) |
| Compute & schedules | Trigger.dev project `finance-engine-v2-jobs` (`proj_wmvfhfcpwnsnfzhrfooh`) |
| Candles + artifacts | R2 bucket `finance-engine-v2` |
| Secrets | project-hub vault, scope `finance-engine-v2` |

## Pipeline

generated → queued → gauntlet (S0 static, S1 novelty/penalty, S2 train, S3 re-tuning walk-forward,
S4 cross-symbol, S5 DSR/permutation/bootstrap, S5b stress, S6 sealed one-shot holdout) →
incubating (≥30 days live paper, kill-switch 5/10/20) → eligible → champion (auto-promote with
archive + rollback; Telegram on every promotion/demotion/halt).

## Schedules (Trigger.dev)

| Task | Cron | Role |
|---|---|---|
| `ingest-candles` | `2 * * * *` | refresh perp klines + funding to R2, validate gaps |
| `paper-step` | `12 * * * *` | hourly paper trade step for champion + incubating |
| `evolve-cycle` | `20 */6 * * *` | GP + LLM generation, queue gauntlets |
| `daily-monitor` | `0 7 * * *` | graduation, auto-promotion, degradation demotion |


## Hard rules

- Paper only. Real execution requires a deliberate connector swap in `src/trigger/paper.ts` — designed for it, not wired.
- Data ≥ `config.sealDate` is sealed: one holdout evaluation per strategy hash, ever (enforced in Convex).
- No strategy skips the gauntlet. There is no manual "just promote it" path that bypasses validation.
